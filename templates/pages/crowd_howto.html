{% trans 'How to start a crowd sourcing project for your collection' %}

Follw the steps below using the Transkribus desktop application to prepare a collection for a crowd sourcing project.

Check list

  • A Transkribus account. Get an account.
  • The Transkribus Desktop appliction. Download.
  • A Collection within Transkribus. Read more.

Preparation phase in Transkribus desktop

  1. Upload of manuscripts
  2. Batch processing of block and line segmentation
  3. Polygon2Baseline tool: open issue in SWTGui (tool is available but needs to be integrated); is needed for beautiful line snippets
  4. Mark as crowdsourcing: can be already done via ‘Edit Collection Metadata’ by owner/editor (may only owner should be allowed to do this). Deselecting means crowdsourcing is closed for the moment. Later on we can handle this automatically with a date field (from/to).
  5. Owner can define a representative collection image and representative image for each doc in the collection to be shown as a thumb image in the browsing interface. As default the first or the image in the middle can be used.
  6. Run HTR (if 2nd crowdsourcing run with a trained model)
  7. Define a tag set for your collection which the WebUI loads and the crowd should use for tagging (will not be available at the start)
